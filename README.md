requirements.txt нужен для того, чтоб можно было командой pip install -r requirements.txt скачивать все библы
разом.

build.spec нужен для внешней утилиты PyInstaller (сборщика ехешника). там хранится всякая инфа для него (потом разберёмся какая).

.ехе программу делаем с помощью PyInstaller. юзаем команду "pyinstaller --onefile --windowed main.py".
"--onefile" собирает всё в один .ехе файл (вместо него можно юзнуть "--onedir", что будет лучше. ниже написано о нём).
"--windowed" убирает чёрное окно консоли при запуске.

У PyInstaller есть два флага. --onedir делает так, что создаётся папка с кучей файлов и ехешником.
--onefile делает так, что создаётся только один бедный ехешник. По моему мнению, --onedir лучше, так как он 
работает эффективнее. --onefile при каждом запуске будет распаковывать себя во временную папку Windows (а ещё антивирусы могут на неё огрызаться).

Чтобы сделать каждому дрону свои айпи и порт, надо зайти по SSH на каждую Radxa и прописать 
статические айпи адреса в сетевых настройках самих Linux систем Radxa
