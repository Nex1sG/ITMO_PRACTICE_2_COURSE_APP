Гипотетическая структура программы

Очевидно, мы будем менять её, пока она не дойдёт до своей почти идеальной формы.


pioneer_telemetry_system/
│
├── main.py                 # Точка входа в приложение (запуск GUI)
├── requirements.txt        # Зависимости (pioneer_sdk2, PyQt6, pyinstaller и т.д.)
├── build.spec              # Конфигурационный файл для сборки в .exe
│
├── config/
│   ├── __init__.py
│   └── settings.py         # Настройки: IP-адреса 4 дронов, порты, пути для сохранения
│
├── backend/
│   ├── __init__.py
│   ├── drone_controller.py # Ваш текущий backend.py (переименован для ясности)
│   ├── async_manager.py    # Класс, управляет 4 дронами одновременно (многопоточность)
│   └── data_logger.py      # Класс, отвечает за сохранение "сырых" данных в файлы
│
├── ui/                     # Папка для графического интерфейса
│   ├── __init__.py
│   ├── main_window.py      # Главное окно приложения
│   └── widgets/            # Кастомные виджеты (например, виджет графика, статус-бар)
│
├── utils/
│   ├── __init__.py
│   └── cli_parser.py       # Ваш текущий parser.py (можно оставить для тестов без GUI)
│
└── logs/                   # Папка, куда будут сохраняться файлы телеметрии (создается автоматически)



.ехе программу делаем с помощью библы pyinstaller. юзаем команду "pyinstaller --onefile --windowed main.py".
"--onefile" собирает всё в один .ехе файл. --windowed убирает чёрное окно консоли при запуске.
